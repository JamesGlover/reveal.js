RecordLoader and Deployment

What's the problem?

  • Lots of PSD 'owned' data
  • Work involves creating records in the database, as well as changing the code
  • Using rake tasks, but easy to forget to run them

Why not use migrations

  • Intent
  • Drift
  • Testability
  • Repeatability
  • Flexibility

The almost solution...

  • rake limber:setup
  • Idempotent
  • Runs on rake application:deploy

    • But...

  • rake application:deploy runs before migrations
  • Complicated

    • rake application:post-deploy

  • Add to psd-deploy in rails role, and enable for supporting apps
  • Hook Idempotent rake tasks into this

    • Keep it simple

    RecordLoader 
    lib/record_loader/plate_type_loader.rb
              
                # frozen_string_literal: true

                module RecordLoader
                  # Creates the specified plate types if they are not present
                  class PlateTypeLoader < RecordLoader::Base
                    self.config_folder = 'plate_types'

                    def create_or_update!(name, options)
                      PlateType.create_with(options).find_or_create_by!(name: name)
                    end
                  end
                end
